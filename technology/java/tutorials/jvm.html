# Java虚拟机（JVM），JDK，JRE和JVM的区别--Java核心知识

Java是高级编程语言。用高级编程语言编写的程序不能直接运行在任何机器上。首先，它需要转换成特定的机器语言。javac编译器就是干这事的，它可以把java程序（包含源码的.java后缀文件）编译成机器认识的代码（就是包含字节码的.class后缀文件）。

那么现在我们知道了JVM的主要功能就是执行编译好的字节码。不同的系统中JVM是不一样的，但是执行字节码得到结果在所有系统中都是一样的。也就是说在Windows中编译得到的字节码在Mac OS中也可以执行，反过来也是一样的。所以java被称为跨平台语言。正如下图所示：

总结：Java虚拟机（JVM）是运行在真实机器（你的计算机）上且可以执行字节码的虚拟机。JVM读不懂源码，因此需要javac编译器将*.java文件编译成*.class文件得到JVM可以执行的字节码。JVM使Java有了可移植性（一次编写，到处运行）。不同的系统中JVM是不一样的，但是执行字节码得到结果在所有系统中都是一样的。

# JVM的结构

我们来看下JVM是如何工作的：

类加载器： 类加载器读取.class文件的内容，然后把字节码保存至方法区

方法区： 在JVM中所有类共享一块方法区，其中存放了每个类的层级信息。

堆： 堆是JVM内存的一部分，实例对象都将存储到这里。JVM将为每个.class文件创建一个类对象。

栈： 栈也是JVM内存的一部分，但它和堆不一样，是用来存储临时变量的。

寄存器： 寄存器将跟踪哪些是执行过的指令和哪些是还未执行的指令。由于指令是由线程执行的，所以每个线程会有多个寄存器。

本地方法栈： 本地方法可以访问虚拟机运行时的数据。

本地方法接口： 本地方法接口使得java代码可以调用本地应用或者被本地应用调用。本地应用则是硬件和操作系统中特定的程序。

垃圾回收机制： java代码将创建类实例，这些实例用完之后会由内存管理的垃圾回收机制自动销毁。

# JVM，JRE和JDK的对比

JRE：JRE是JVM的运行环境。JRE包含JVM，类库和其他文件，但是不包含一些开发工具，如编译器和调试器。所以你可以在JRE中运行代码但是不可以在JRE中开发和编译代码。

JVM：正如上面所提及的，可以使用JRE提供的类，库和一些文件来运行程序。

JDK: JDK是JRE的超集。JDK包含JRE的所有以及其他开发工具，如编译器，调试器等等。
